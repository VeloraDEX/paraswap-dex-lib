# Resolved Build Fixtures

These JSON files are generated by `yarn fixtures:generate` and checked by
`yarn fixtures:check`.

Do not hand-edit `expectedParams`, `expectedTx`, or negative `expectedError`
values as the primary workflow. Change the deterministic case definitions in
`../resolved-build-fixture-cases.ts`, regenerate the fixtures, and review the
resulting JSON diff.

Negative fixture `expectedError` values are auto-derived from the current
boundary validation behavior. A changed error string should appear as a fixture
diff after regeneration.

For a schema-version bump, update `RESOLVED_BUILD_SCHEMA_VERSION`, add a
migration that rewrites every committed JSON fixture, regenerate, and update the
schema-version rejection test.
